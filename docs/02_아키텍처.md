# 시스템 아키텍처

## 구성도
```
[현장 작업자 폰]            [관리자 PC]
  모바일 웹(PWA)             웹 대시보드(React)
   촬영·업로드                지도·목록·통계
        \                      /
         \                    /
          v                  v
        ┌───────────────────────┐
        │   AI 추론 서버 (FastAPI) │
        │  - 이미지 업로드 저장     │
        │  - YOLO 균열 탐지         │
        │  - 위험도 산정 엔진       │
        │  - SQLite 점검 이력 저장  │
        └───────────────────────┘
                  │
            best.pt (Colab에서 학습)
```

## 기술 스택 (프로토타입)
- **AI 서버**: Python, FastAPI, Ultralytics YOLO, SQLite
- **프론트엔드**: React + Vite (현장 촬영 + 관리자 대시보드 한 앱)
- **모델 학습**: Google Colab (무료 GPU)
- **지도/차트**: Leaflet, Recharts

> 실서비스 확장 시: PostgreSQL+PostGIS, 객체스토리지(S3), 작업큐(Redis/Celery), Flutter 네이티브 앱.

## API 명세 (핵심)
| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/inspections` | 이미지+위치 업로드 → 분석 실행 → 결과 반환 |
| GET | `/api/inspections` | 점검 목록 (필터: 위험등급) |
| GET | `/api/inspections/{id}` | 점검 상세 (탐지 결과 포함) |
| GET | `/api/inspections/{id}/image` | 결과 시각화 이미지 |
| GET | `/api/stats` | 통계 (등급 분포, 결함 종류 분포) |
| GET | `/api/facilities` | 시설물 목록 + 최신 위험도 |

## 데이터 모델 (SQLite)
- **facility**: id, name, type(교량/터널/...), lat, lng
- **inspection**: id, facility_id, image_path, result_image_path, created_at, risk_grade, risk_score, defect_count
- **detection**: id, inspection_id, label, confidence, bbox(x,y,w,h), width_px

## 위험도 등급
A(양호) · B(주의관찰) · C(보수필요) · D(긴급보수) · E(사용제한)
산정식: 결함종류 심각도 + 균열 폭 + 결함 밀도 + 개수의 가중합.
