# 2개월 실행 로드맵

> 목표: 발표 가능한 **작동하는 프로토타입** = 현장 촬영(모바일 웹) → AI 분석 → 위험도 → 관리자 대시보드 전체 흐름.

핵심 원칙: **AI 서버는 모델이 없어도 mock 모드로 즉시 동작**한다. 그래서 전체 시스템을 먼저 완성하고, 진짜 모델을 나중에 끼운다. (병렬 진행 가능)

## 주차별 계획

| 주차 | 목표 | 산출물 |
|------|------|--------|
| 1주 | 환경 세팅 + 전체 뼈대 실행 | AI서버 mock 동작, 웹 대시보드 화면 뜸 |
| 2주 | Colab 균열 탐지 첫 학습 성공 | `best.pt` (소형 공개 데이터) |
| 3주 | 진짜 모델을 AI 서버에 연결 | 업로드한 사진에서 실제 균열 탐지 |
| 4주 | 위험도 엔진 튜닝 + DB 연동 | 위험등급 A~E, 점검 이력 저장 |
| 5주 | 모바일 촬영 화면 완성 | 폰으로 촬영→업로드→결과 확인 |
| 6주 | 관리자 대시보드 완성 | 지도/목록/통계/보수 우선순위 |
| 7주 | AI-Hub 데이터로 모델 고도화 | 다중 결함 10종 탐지 모델 |
| 8주 | 통합 테스트 + 발표 자료 | 데모 시나리오, 발표 PPT, 보고서 |

## 컴퓨팅 예산 계획 (Colab, 약 30만원)
- 학습은 Colab GPU 사용 (내 PC RX 7600은 개발·추론만).
- **1단계 무료 T4**: 파이프라인 검증(filekey 1개 + `--limit 3000`). 0원.
- **2단계 Colab Pro+** (약 7만원/월 × 1~2): 백그라운드 실행(브라우저 꺼도 24h 학습), 본 학습.
- **3단계 A100/컴퓨팅 단위 충전** (남은 예산): 막판 대규모(수만 장) 고도화.
- GPU 시간당 컴퓨팅 단위 ≈ T4 2 / L4 5 / A100 13.
- 결론: 30만원이면 2달 Pro+ + A100 대규모 학습까지 커버 가능.

## 병렬 트랙 (시간 절약)
- **트랙 A (AI)**: Colab 학습 — AI-Hub 데이터 받고 학습 (백그라운드로 오래 걸림)
- **트랙 B (시스템)**: 서버·웹·모바일 — mock 모드로 먼저 완성 (이미 완료)

두 트랙을 동시에 진행하다가 3주차에 합류시킨다.

## 위험 요소 & 대비
- **데이터 규모(118GB)**: 전체 대신 filekey 일부 + `--limit`으로 부분 학습 → 점진 확대.
- **AMD GPU 학습 불가**: 학습은 Colab GPU 사용(예산 확보됨). 내 PC는 개발·추론만.
- **Python 3.14 호환성**: AI 서버는 Python 3.12 가상환경 사용 (설치 완료).

## 최소 성공 기준 (MVP)
1. 사진을 올리면 균열 위치가 박스로 표시된다.
2. 위험등급(A~E)과 점수가 나온다.
3. 관리자 화면에서 점검 목록과 보수 우선순위를 본다.

## 성능 목표 (공식 베이스라인 대비 — `docs/03_데이터셋_핵심정리.md` 참고)
- 공식: CvT 분류 F1 **84.67%**, SegFormer 분할 mIoU **55.78%**
- 우리(YOLOv11-seg 단일 모델): 위 수치를 비교 기준으로 삼아 mAP50 / mIoU 보고.
- 데이터 분할은 공식과 동일하게 **80/10/10**(train/val/test) 권장.
